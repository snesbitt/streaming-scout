How this system earns the trust to recommend anything

Six steps, in sequence. Each stands on its own — accountability doesn't depend on finishing the rest. Built to be read by anyone, technical or not: every box below is a real part of the system, not an illustration of an idealized one.

Field note

The first deploy ranked a generic trending chart above your actual taste — a slapstick comedy nearly outscored a spy thriller with Rami Malek, because every genre carried equal weight and every candidate came from the same undifferentiated "trending now" list. Step 03 exists because a recommendation is only as trustworthy as the evidence behind it.

01Tools & mode

Built entirely in Cowork — no terminal, by design. The Chrome extension reads your already-authenticated Netflix and Prime sessions directly; no credential is ever requested, stored, or transmitted. Trust in a system starts with what it refuses to collect.

02No metadata — so it gets researched

Netflix and Prime's own history pages disclose a title and a date — nothing else. Every genre tag and cast credit across the full 190-title record was independently sourced through per-title research, not inferred from platform metadata. Watch status is likewise inferred from episode counts and gaps, never presented as scraped fact. Coming Soon draws only from named, verifiable sources — BritBox, PBS Masterpiece, Netflix Tudum, Den of Geek, What's on Netflix, JustWatch — cited so each claim's reliability can be judged independently rather than simply trusted. Where verified art doesn't exist, the record says so plainly by leaving the space empty, rather than substituting a guess or a placeholder that looks like it might be real.

03Signature genres in full, generic at a third

Broad categories — Comedy, Drama — are shared across too many unrelated titles to constitute meaningful evidence on their own. Genres with real recurrence in your own history (British Procedural, Spy Thriller, Mystery, Action Thriller) are weighted in full, with an added signal for cast overlap against titles you've already valued. A genre you've explicitly rejected is capped regardless of any other data pointing the other way — expressed preference overrides inferred pattern.

Full weight

British Procedural, Spy Thriller, Mystery, Action Thriller — genres with real recurrence in your own watch history, plus cast-overlap bonus.

Capped, regardless of score

Sci-fi, fantasy, rom-com — explicitly rejected. A high score elsewhere can't override a genre you've already ruled out.

04A snapshot, not a live connection

As with the other two systems in this portfolio, this one reads from a single published snapshot rather than a live session — nothing touches your Netflix or Prime account while you're using it. Synchronization happens deliberately, in a Cowork session, either at your request or on a fixed Monday schedule, and only then is the static record rebuilt.

05Deploy through Netlify's upload page — no Functions yet

This environment blocks direct calls to Netlify's API and CLI — confirmed empirically by a 403 on every Netlify domain attempted, not assumed. The functioning workaround is Netlify's own upload interface, driven through the Chrome extension in place of a terminal; the Monday scheduled run uses the identical path. That path is limited to static files: Netlify Functions, which power the backend of the other two systems, require continuous deployment from a connected Git repository — verified both by a direct functional test and by Netlify's own documentation. Closing that gap means linking this project to GitHub, the same governance model the other two already operate under.

06What's still manual

Dismissing a title updates your device immediately, but making that exclusion permanent still requires your confirmation in chat — there is no backend authority yet to write EXCLUDED_TITLES.md independently. Equally absent: a health check verifying that the Monday run actually published cleanly. Until both exist, a glance at the live site after each scheduled run remains the control of record.

What this took to learn

A recommendation is only as honest as its candidate pool — sourcing from a generic "trending now" list regresses to generic results no matter how the scoring is built.

Authorization precedes any expenditure. Even a domain name required its exact identity and cost confirmed before purchase, and entering payment remained a decision left to you, not delegated.
